¿Qué pasa cuando se inicializa un repositorio? ¿Por qué necesita para hacerlo?

	git crea los archivos necesario para empezar a llevar el seguimiento
	
¿Cómo es el área de ensayo diferente del directorio de trabajo y el repositorio? ¿Qué valor crees que se ofrece?

	es como un espacio logico donde le decimos a git q archivos vamos 
	tener en cuenta par el siguiente commit. permite hacer varios 
	cambios logicos y luego hacer un commit por cada uno.

¿Cómo puede usar el área de ensayo para asegurarse de que tiene una cometen por cambio lógico?

	puedo verificar las diferencias entre el directorio de trabajo y el
	area de ensayo, o entre esta ultima con respecto al ultimo commit

¿Cuáles son algunas situaciones en las ramas serían útiles para mantener su historial organizado? ¿Cómo ayudar a las ramas?

	para exprimentar o crear versiones diferentes, sin comprometer la 
	version estable

¿Cómo los diagramas ayudan a visualizar la estructura de las ramas?

	nos permite conocer la difurcacion que tinene nuestras versiones

¿Cuál es el resultado de la fusión de dos ramas juntos? ¿Por qué lo representamos en el diagrama de la forma en que hacemos?

	el resultado es una nueva rama con los commit de ambos y los archivos
	fucionados, se representan para facilitar su comprencion

¿Cuáles son los pros y los contras de frente a la combinación automática de Git siempre haciendo fusiones de forma manual?

	la ventaja es el control sobre el merge la contra es lo tedioso
	que puede llegar a ser
